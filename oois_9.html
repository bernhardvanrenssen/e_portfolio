Collaborative Discussion 2: Alternatives to SQL

Unit 8 - 10

Initial Post:

Not Only SQL (or NoSQL) is a non-relational database management system (DBMS) that derived from the traditional SQL or relational database system. Unlike its predecessor, NoSQL databases is much more scalable, as it does not store its data in relational tables – i.e. its data is stored in a non-normalized way. With large social-media companies having a need for fast access to a huge amount of data, NoSQL proves to be a much faster database by storing large data entities using documents in formats like XML or JSON (Sharma & Dave, 2012).

One of the most popular and accepted NoSQL developments is the Key-Value database. Under this database, A Key value is assigned as a string value and a Value for each Key provided as a string, JSON/XML value. The Key is used as the index when searched for, and the Value then returned, much like a relational database, but with each table only containing two columns.

It is widely popular due to its simplicity, scalability and flexibility (due to the fact that it can include JSON/XML files). It is a relevant and fast-growing database model and often used in the gaming industry or websites that require a large amount of caching. It does however have its limitations, as there are no primary/foreign keys for relationships between tables, and thus not ideal for operations other than CRUD (Create, Read, Update, Delete) (Rajani, 2020).

Other typical NoSQL databases include Document Stores that functions similar to the Key-Value database, but handles large amounts of data; Wide Column databases that provides a semi-structured database which stores its data in sections of columns rather than data rows, and Graph databases which contains index-free data by having a node point to the next item, ideal if traversing through data is required (Ali, et al., 2019).

NoSQL provides different approaches to data storage, each with their own pros and cons. But with the need for Big Data ever expanding, new and improved ways to manage large datasets faster are in high demand – highlighting the increased need for NoSQL DBMS.

Peer Response (Kaoru Kitamura):

From my point of view, I think the most useful aspect of a NoSQL database is its flexibility. This enables developers to manipulate data more easily than storing into a traditional RDB.

As posted above, JSON or XML formats can be stored into NoSQL databases such as MongoDB. It seems to me that this is one the most useful aspects of using NoSQL databases. Usually, developers have to transform data into normalized tables' structure when inserting data objects in applications into relational databases. This can lead to the enhancement of development speed, even cost reduction of implementation.
