What is the (theoretical) maximum number of disks that your program can move without generating an error?

The theoretical maximum number of disks that this function can solve is 999. When prompted to enter a number of disks, anything larger than 999 disks will throw a Runtime Exception with error: “Maximum Recursion Depth exceeded”. This can however be altered as python sets this recursion limit for functions to 10^4 by default to prevent a stack overflow.

What limits the number of iterations? What is the implication for application and system security?

The limit of recursive functions is limited by default by the “sys” module in python, specifically the setrecursionlimit() function. The program then will crash if any number larger than the set limit (10^3) is entered for a recursive function. This can however manually be overridden, but increases the chances of generating a stack overflow.
