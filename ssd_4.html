Q1: What factors determine whether a programming language is secure or not?

A secure architecture of system should include aspects of confidentiality, integrity and availability (Pillai, 2017). Faroog et. al. (2014) suggested that the following parameters can be considered to evaluate a language for its strength in security:

  • language should avoid dangling references
  • there should not be any memory leakage
  • control over array index out of bound
  • pointers arithmetic
  • prevent stack and heap overflows

According to OWASP top 10 cyber security risks, new and improved attacks are always developed by mindful cyber pirates (OWASP, 2022). A big security consideration would thus be the popularity of the language, as well as the availability of up-to-date libraries that are used within the application. Outdated or less popular languages are often not updated against newer attacks and threats, and thus pose a bigger security threat. A language like python, that consists of almost 20% of all github pull requests, is thus a good and safe bet as a secure language (Faizan, 2021). But as in any programming language, a major consideration would be the developer’s efficiency with the language, and its inherent security features.

Q2: Could Python be classed as a secure language? Justify your answer.

I think Python can be classed as a secure language. Although Python has quite a few vulnerabilities, there are strategies that can be applied to mitigate the security issues.

For example, libraries can be used to mitigate risks of handling passwords and sensitive information as well as race conditions (Pillai, 2017).

Besides, as Python is a popular language, there are projects (eg. Python Open Web Application Security Project) working on creating a hardened version of Python which will be more resilient to security threats than the standard CPython (Pillai, 2017).

As in any language, the level of security is to a large degree up to the developer using the language. You can do very secure things in any language, as well as very insecure things. Due to its popularity though, python does seemingly have a lot of secure libraries that are widely used, and well-maintained, meaning up-to-date for protection against the latest cyber threats.

I must add, that since Google, NASA, Netflix and Facebook primarily use and trust Python as their programming language, it speaks a great deal of confidence into using it as a secure language (Foong, 2020)

Q3: Python would be a better language to create operating systems than C. Discuss.

C is the foundation of Python. However, C is compiled while Python is interpreted. Based on this difference, every change made in the C source code needs to be compiled before being able to execute. Frequent changes may result in a vast amount of waiting time. On the other hand, Python works by interpreter, which only needs to translate the affected code and results in less waiting time. In terms of development and editing, Python may be a better choice for speedy development.

However, interpreted programs run slower than complied programs, known as “interpretive overhead”. When selecting a language for system development, there is no absolute best language, but rather a compromise among different factors.
